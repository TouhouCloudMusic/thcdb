# User Lists (用户列表) 模块

> **实现状态**: ❌ 未实现 | [查看路线图](../ROADMAP.md#user-list)

用户列表允许用户创建音乐实体的自定义收藏，用于个人组织和分享。

## 系统概述

用户列表系统包含两个主要实体：

- **user_list**: 包含元数据的列表容器
- **user_list_item**: 列表中的单个项目

## UserList 实体

用户列表包含以下信息：

- **名称**: 列表的显示名称
- **描述**: 列表用途的详细描述
- **所有者**: 创建列表的用户
- **可见性**: 列表是否公开可见

### 可见性设置

- **公开**: 对所有用户可见
- **私有**: 仅对创建者可见

## UserListItem 实体

列表中的单个项目：

- **实体引用**: 被引用实体的类型和标识
- **描述**: 此项目的可选自定义描述

### 可引用的实体

列表项目可以引用各种实体类型，见 [EntityType](../shared-types.md#entitytype)。

## 使用场景

### 个人组织

- **收藏夹**: 喜爱的歌曲或发行版本收藏
- **播放列表**: 精心策划的聆听序列
- **愿望清单**: 待获取或探索的项目
- **主题收藏**: 主题分组（如"东方人声编曲"）

### 社区分享

- **推荐列表**: 与其他用户分享的精选列表
- **协作列表**: 社区维护的收藏
- **发现列表**: 突出较少知名作品的列表

## 隐私和权限

### 访问控制

- 列表创建者对其列表拥有完全控制权
- 公开列表对所有用户可读
- 私有列表仅创建者可访问
